Rmarkdown offers a way to combine plain text and results from your analysis in a powerful way. The output can be a PDF, static webpage (HTML document) or a Word doc. There are many ways to customize it but here Ill just present the very basics.

One of the features of markdown is that you can format plain text easily without the need of knowing more complex (albeit more powerful) markup languages.

Rmarkdown interprets each line as simple text unless it encounters a chunk of code. A chunk of code is surrounded by

```{r}
#Your R code goes here
library(tidyverse)
plot(cars)
```

Next, is how to format your text. Markdown is quite elegant in the way it handles the formatting. Surround text between two underscores _ to make it italics or between double asterisks ** to bold . You can even combine both. To make a list just add an asterisk to the start of each line:

  • **bold** = bold
  • _italics_ = italics
  • _**both**_ = both

Also works for numbered lists

  1. Uno
  2. Dos
  3. Tres
  4. Catorce

(^ you can also include links to websites)

To convert your Rmarkdown file to a nicely HTML or PDF, click the Knit button below the scripts tab (top border in RStudio). All your text and R code will be run together to produce the output.

All of your Rcode has to be exactly the same as in a normal R script. You need ot load the libraries

library(tidyverse)

Check the contents of data:

head(mpg)

You can also add plots:

ggplot(mpg,aes(x=cty,y=displ,color=drv)) +
  geom_point() + 
  geom_smooth() +
  theme_light()

You can have finer control of the behaviour of the chunks. Adding parameters to the {r} portion you can. For example, adding echo=F and message=F wont output the code that produced the plot or any messages (like the ones geom_smooth() produces):
